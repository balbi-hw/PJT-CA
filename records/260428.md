Spring 을 활용해 처음으로 웹 어플리케이션을 만들어봤다. 강의를 따라하며 몇 개 만들어본적은 있지만 내가 주도해서 뭔가를 만든 건 처음이라 기쁘기도 하지만 쉽지만은 않았기에 앞으로의 여정이 걱정된다.
  
이번에는 MVP 로 최소기능만 구현했다. 사실 CRUD 의 UD 가 없어서 최소기능도 아닌 것 같기는 하지만 뭐.. 그래도 만들긴 했다.
  
만들면서 힘들었던 점은 다음과 같다.

- 연관관계 매핑:  
강의에서는 몇 번 해봤는데 능동적으로 매핑을 진행하려하니 조금 헤맨 부분이 있다. 하지만 계속하면 나아질 수 있을 것 같다.
  
- DTO 개념의 이해:  
이론을 공부하며 DTO 에 대한 내용이 잘 이해되지 않았는데 이번 기회에 완벽히 이해했다. 웹 구현까지 마친 뒤 `Deck` 엔티티의 `size` 필드가 딱히 필요해보이지 않아서 지웠더니 에러가 발생했다. 왜 그런가 살펴보니 API 로 DTO 가 아닌 엔티티를 넘기고 html 에서 로직을 이용해 데이터를 처리했더니 엔티티 필드에 변화가 생기자마자 API 스펙이 변해버린 것이다. html 에서는 처음 제공해주었던 `Deck.size` 같은 코드를 통해 데이터를 받고 있는데 size 필드를 없애버리니 빌딩이 안되는 것이다.  
DTO 를 만들어서 넘기게 되면 필드에 변화가 생겨도 DTO 스펙에 맞추어 데이터를 가공해주면 문제가 생기지 않는다. 완전히 자유로워지는 것은 아니지만 엔티티를 넘기는 방식보다는 안전하다.  
이를 통해 프로그램 설계가 왜 중요한지 조금이나마 알게 된 기분이다.
  
이외에도 DI 방법이나 Dirty checking, entitymanager 의 merge 메서드 등 몇 가지 더 있지만 아직 설명할 수 있는 정도는 아닌 것 같아 추후 더 공부하고 블로그에 업로드 해야겠다.
