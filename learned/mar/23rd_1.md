### 1. 실행 흐름 이해
1. 입력을 받는다.
2. 조건을 검사한다.
3. 필요한 계산 / 처리를 한다.
4. 결과를 출력한다.

```java
public class Main {
    static void main(String[] args) {
        System.out.println("프로그램 시작");
        System.out.println("작업 수행");
        System.out.println("프로그램 종료");
    }
}
```
> 흐름은 알고리즘 문제와 결국 크게 다르지 않구나. 생각해보면 사실 둘 다 프로그램이니까 당연하다.

### 3. 조건문과 반복문
> 문법은 알고 있지만 다음의 내용은 기억할만하다. 하지만 이 또한 알고리즘 문제와 비슷하다.

- 프로그램의 판단 로직 (if)
  - 회원이면 허용
  - 비회원이면 거절
  - 비밀번호가 맞으면 로그인
  - 중복 이메일이면 가입 불가

- 반복문
  - 회원 목록 전체 조회
  - 게시글 여러 개 출력
  - 배열 / 리스트 전체 탐색
  - 점수 합계 계산

### 4. 메서드 분리
문법보다 코드를 기능별로 나누는 것이 중요하다.
> OOP 에서도 중요한 개념이라고 생각하는 부분이다.
  
아래 코드를 리팩터링 해보자
```java
public class Main {
    public static void main(String[] args) {
        int[] scores = {80, 90, 100, 70, 60};

        int sum = 0;
        for (int i = 0; i < scores.length; i++) {
            sum += scores[i];
        }

        double avg = (double) sum / scores.length;

        System.out.println("총합: " + sum);
        System.out.println("평균: " + avg);
    }
}
```
```java
public class Main {
    public static void main(String[] args) {
        int[] scores = {80, 90, 100, 70, 60};
        
        int sum = getSum(scores);
        double avg = getAvg(scores);

        System.out.println("총합: " + sum);
        System.out.println("평균: " + avg);
    }
    
    public int getSum(int[] array) {
        int sum = 0;
        for (int nums : array) {
            sum += nums;
        }
        return sum;
    }
    
    public double getAvg(int[] array) {
        int sum = getSum(array);
        return (double) sum / array.length;
    }
}
```
합계와 평균을 구하는 기능을 메서드로 분리해냈다.
  
메서드는 단순히 코드를 줄이는 도구가 아니다.
- 기능에 이름을 붙인다.
- 역할을 분리한다.
- 재사용할 수 있다.
- 가독성이 좋아진다.

> 추후 스프링에서 메서드 분리 감각이 클래스 분리, 서비스 분리, 컨트롤러 분리로 이어진다고 하니
> 평소 알고리즘을 풀 때도 연습하자.

## 실습
- 메뉴형 프로그램 만들기

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);

        while (true) {
            System.out.println("1. 자기소개 출력");
            System.out.println("2. 두 수 더하기");
            System.out.println("3. 종료");

            System.out.print("메뉴를 선택하세요: ");
            int num = input.nextInt();
            input.nextLine();

            if (num == 1) {
                introduce();
            } else if (num == 2) {
                System.out.println(add());
            } else if (num == 3) {
                System.out.println("프로그램을 종료합니다.");
                break;
            } else {
                System.out.println("잘못된 입력입니다.");
            }
        }

    }
    public static void introduce() {
        System.out.println("안녕하세요? 저는 이현우라고 합니다.");
    }

    public static int add() {
        Scanner input = new Scanner(System.in);

        System.out.print("첫 번째 숫자를 입력하세요: ");
        int first = input.nextInt();
        input.nextLine();

        System.out.print("두 번째 숫자를 입력하세요: ");
        int second = input.nextInt();
        input.nextLine();
        return first + second;
    }
}
```
> 여러번 만들어봤던 프로그램이지만 인풋을 메서드 안에서 받게 만들어뒀더니 메서드 안에서도 스캐너를 정의해줘야했다.  
> 생각해보면 당연한건데 아직 익숙치 않은 듯 하다.
